The demo will follow the content in _https://github.com/ajstamm/shiny-a11y-app/tree/master/presentation%20scripts/RMedicine%202026_

* Introduction (~8 minutes)
    * Who we are (1 minute)
    * Where to download files (1 minute)
    * Tour of a basic dashboard (2 minutes)
    * Tools used to test for accessibility (2 minutes)
        * NVDA, ANDI, `shinya11y`
    * Accessibility checks the dashboard failed (2 minutes)
* Screen reader related checks (~10 minutes)
    * Rethinking visual content (1 minute)
    * The only way to access data in the charts is by looking at them (4 minutes)
        * Test, address, retest
    * There is no alternative text for charts (4 minutes)
        * Test, address, retest
    * The dashboard's language is not defined (1 minute)
        * Test, address, retest
* Keyboard related checks (~9 minutes)
    * Rethinking slider use (1 minute)
    * Both sliders lack keyboard focus (3 minutes)
        * Test, address, retest
    * The double-headed slider does not work with the keyboard (5 minutes)
        * Test, address, retest
* Understanding and clarity related checks (~19 minutes)
    * Rethinking navigation design (1 minute)
    * The dashboard does not have a browser tab title (1 minute)
        * Test, address, retest
    * There is only one way to navigate around the dashboard (5 minutes)
        * Test, address, retest
    * There are no navigation instructions (3 minutes)
        * Test, address, retest
    * There are no descriptions or explanations for the charts (3 minutes)
        * Test, address, retest
    * The filters are labelled only with vague keywords (3 minutes)
        * Test, address, retest
    * The filters display for all tabs, even where they are not relevant (3 minutes)
        * Test, address, retest
* Conclusion (~2 minutes)
    * Consider accessibility in dashboard design (2 minutes)
* Questions (~10 minutes)
